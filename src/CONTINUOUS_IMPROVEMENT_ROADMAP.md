## 🔄 IMPROVEMENT PROCESS

### **Identify:**
```
Sources:
- User feedback
- Support tickets
- Analytics data
- Error logs
- Team observations
- A/B test results

Weekly: Compile improvement opportunities
Priority: Impact × Effort matrix
```

---

### **Prioritize:**
```
Framework: RICE Score
R - Reach (how many users affected?)
I - Impact (how much improvement?)
C - Confidence (how sure are we?)
E - Effort (how much work?)

RICE = (R × I × C) / E

Top 5 RICE scores → Sprint backlog
```

---

### **Implement:**
```
Process:
1. Write spec/design (1-2 days)
2. Review with team
3. Implement & test (2-5 days)
4. Code review
5. Deploy to staging
6. QA testing
7. Deploy to production
8. Monitor impact
```

---

### **Measure:**
```
For each improvement:
- Define success metrics
- Set baseline
- Set target
- Measure after 1 week
- Measure after 1 month
- Document learnings
```

---

### **Iterate:**
```
Results:
- Success? → Document and share
- Partial? → Iterate and improve
- Failure? → Learn and pivot

Always: Extract learnings
Always: Update documentation
Always: Share with team
```
